## Current Results

### CLAHE Preprocessing

Synthetic test source:

- Synthetic 480x360 lighting samples generated by `python-scripts/create_sample_images.py`
- Not a real biometric dataset
- Purpose: verify the preprocessing workflow and evidence generation

Command:

```powershell
C:\Users\deshm\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe python-scripts\clahe_preprocess.py --input samples\lighting --output assets\evidence\clahe --csv benchmarks\results\clahe_timings.csv
```

Latest synthetic measured result:

- 5 synthetic images processed
- Average CLAHE preprocessing time: about 15.36 ms
- Output evidence folder: `assets/evidence/clahe/`
- Timing CSV: `benchmarks/results/clahe_timings.csv`

Consented real-image test source:

- 3 real test photos provided by the project owner for development evidence
- Purpose: check preprocessing behavior on realistic selfie images with indoor light, shadow, and backlight
- Public evidence folder: `assets/evidence/real_clahe/`
- Timing CSV: `benchmarks/results/real_clahe_timings.csv`
- Average CLAHE preprocessing time: about 81.93 ms on full-size images

Important note:

These are Python development script results, not final mobile runtime results. Final mobile CLAHE should run after face crop/resize and should be implemented using native C++ or another optimized mobile path, then measured separately.
